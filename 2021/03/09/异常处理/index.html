异常处理

Java 异常处理

1. 异常分类

如果，Error 类层次描述了Java运行时系统的内部错误和资源耗尽错误。这种内部错误，除了通知用户并尽力妥善地终止程序外，你几乎无能为力，这种情况很少出现。

重点关注 Exception 层次结构。两个分支 RuntimeException 表示由编程错误导致的异常，如果程序没有问题，但由于像I/O错误这类问题导致的异常。

RuntimeException 的异常包括以下问题：

  • 错误的强制类型转换。
  • 数组访问越界。
  • 访问null指针

不是 RuntimeException 的异常包括:

  • 试图超越文件末尾继续读取数据。
  • 试图打开一个不存在的文件。
  • 试图根据给点的字符串查找Class对象，而这个字符串表示的类并不存在。

2. 抛出异常

如果遇到了无法处理的情况，Java方法可以抛出一个一个异常。方法不仅告诉编译器要返回什么值，话要告诉编译器将要返回什么值。比如读取文件时，抛出 FileNotFoundStream 异常可以处理文件无法找到的情况。

抛出异常的一个简单的例子

    public static int divide(int a, int b) throws Exception{
        if(b==0){
            throw new Exception("除数不能为0");
        }else{
            return a/b;
        }

    }

    public static void main(String[] args) throws Exception {
        divide(1,0);
        System.out.println("continue");
    }

结果是：

    Exception in thread "main" java.lang.Exception: 除数不能为0

我们发现这个并没有打印接下来的，抛出异常后就直接结束的程序，这也说明如果能自己解决的问题，如果抛出异常导致中止程序反而得不偿失。

在实际情况中，我们往往有很多属性是Java内置 Exception 所预料不到的，但是确实是不应该发生的，这时候我们应该自己定义异常。

举个例子:

    // GenderException.java
    public class GenderException extends Exception{// 继承Expection的性别异常
        public GenderException(String msg){
            super(msg);
        }
    }


    // Person.java
    public class Person {
        private final String gender;
        private final String name;
        public Person(String g, String n){
            gender = g;
            name = n;
        }
        String getGender(){
            return gender;
        }
        String getName(){
            return name;
        }
    }

    
    // WashRoom.java
    public class WashRoom {
    
        public void woman(Person p) throws GenderException{
            if(p.getGender().equals("女")){
                System.out.println("欢迎光临");
            }else{
                throw new GenderException("男士止步！");
            }
        }
    }


    // test.java
    public class test {
        public static void main(String[] args) throws GenderException {
            Person XiaoMing = new Person("男", "小明");
            WashRoom washRoom = new WashRoom();
            washRoom.woman(XiaoMing);
        }
    }

结果：

    Exception in thread "main" test.wash.GenderException: 男士止步！

就抛出了自己定义的错误。

3. 捕捉异常

我们来看一个简单的捕捉异常

    public static void main(String[] args) {
        try{
            System.out.println(1/0);
        }catch (Exception e){
            e.printStackTrace();// 打印异常信息
        }finally {
            System.out.println("finally");
        }
        System.out.println("continue");
    }

结果是:

    java.lang.ArithmeticException: / by zero
        at test.Catch.test.main(test.java:7)
    finally
    continue

我们看到它打印了异常信息，也打印出了接下来的内容。要注意，虽然从打印的内容来看，finally 可有可无，实际上，finally 语句块是不论异常是否发生，都会执行的，也起到了释放 try 资源的作用。

我们也发现，捕捉异常的话，代码是可以继续运行的，所以为了线程能持续运行，在不危害数据的情况下，尽量去捕捉异常，可以让进程继续下去。

在 catch 语句中，用 Exception 可以捕捉所有的异常，也可以捕捉单个异常，包括自己写的异常，一个 try 语句可以匹配多个异常，也可以在catch中再抛出异常，形成异常链，终止程序(finally仍会执行)。

try-with-Resources语句

带有资源的try语句

try(Resource res = ...)
{
    work with res
}

try结束会自动调用 res.close() ，不论是否触发异常。
