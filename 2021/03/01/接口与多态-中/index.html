接口与多态(中)

接口与多态(中)

让我们来看看常见而实用的接口，下面的接口一定要理解并弄明白。

1. Serializable

一个对象序列化的接口，一个类只有实现了Serializable接口，它的对象才能被序列化。

什么是序列化？

序列化是将对象状态转换为可保持或传输的格式的过程。与序列化相对的是反序列化，它将流转换为对象。这两个过程结合起来，可以轻松地存储和传输数据。

当我们需要把对象的状态信息通过网络进行传输，或者需要将对象的状态信息持久化，以便将来使用时都需要把对象进行序列化。

Serializable 在源码上也只是一个声明，就是告知JVM，让JVM帮助这个类进行序列化，而自己不用具体去写方法。

为什么要定义serialversionUID变量

如果我们标识了 Serializable 却没有具体实现，接口会默认生成一个 serialVersionUID 变量，但是接口端的 serialVersionUID 异常敏感，反序列化可能导致异常，所以建议自己定义一个 serialVersionUID。

下面看一段示例

public class Employee implements Serializable {
    private static final long serialVersionUID = 1L;
    private int age;
    private int salary;
    private String name;

    public void setAge(int age) {
        this.age = age;
    }

    public void setSalary(int salary) {
        this.salary = salary;
    }

    public void setName(String name) {
        this.name = name;
    }

    @Override
    public String toString() {
        return "Employee{" +
                "age=" + age +
                ", salary=" + salary +
                ", name='" + name + '\'' +
                '}';
    }
}


public class Main {
    public static void writeEmployee(){
        try{

            Employee em = new Employee();

            em.setAge(18);

            em.setName("abc");

            em.setSalary(10000);


            ObjectOutputStream objectOutputStream = new ObjectOutputStream(new FileOutputStream(new File("abc")));

            objectOutputStream.writeObject(em);
            System.out.println("序列化成功");
            objectOutputStream.close();
        } catch (Exception ex) {
            System.out.println("WriteEmployee" + ex.getMessage());
        }
    }

    private static Employee readEmployee(){
        Employee me = null;

        try{
            ObjectInputStream objectInputStream = new ObjectInputStream(new FileInputStream(new File("abc")));

            me = (Employee) objectInputStream.readObject();

            System.out.println("反序列化成功");

            System.out.println(me);
        }catch (Exception ex){
            System.out.println("readEmployee:"+ex.getMessage());
        }
        return me;
    }

    public static void main(String[] args) throws Exception {
        writeEmployee();
        readEmployee();
    }
}

结果

序列化成功
反序列化成功
Employee{age=18, salary=10000, name='abc'}

如果没 serialversionUID 在反序列化时候就回抛出异常，这个 serialVersionUID 的详细的工作机制是：在序列化的时候系统将 serialVersionUID 写入到序列化的文件中去，当反序列化的时候系统会先去检测文件中的 serialVersionUID 是否跟当前的文件的 serialVersionUID 是否一致，如果一直则反序列化成功，否则就说明当前类跟序列化后的类发生了变化，比如是成员变量的数量或者是类型发生了变化，那么在反序列化时就会发生 crash ，并且回报出错误。

2. Comparable 和 Comparator

顾名思义咯，俩兄弟都是比较两个对象用的，要的是学习两个独特的用法和区别。

Comparable

先看源码

public interface Comparable<T> {
    int compareTo(T t);
}

所以如果一个类实现了 Comparable 接口，一定需要重写 compareTo 方法，用自己给定的规则对两个创建的对象进行比较。

示例

public class Employee implements Comparable<Employee> {
    private static final long serialVersionUID = 1L;
    private int age;
    private int salary;
    private String name;

    public Employee(int age, int salary, String name) {
        this.age = age;
        this.salary = salary;
        this.name = name;
    }

    @Override
    public int compareTo(Employee o) {
        return this.salary - o.salary;
    }

    public static void main(String[] args) {
        Employee a = new Employee(18, 100, "A");
        Employee b = new Employee(18, 101, "B");

        if(a.compareTo(b)>0){
            System.out.println(a.name+"工资高");
        }else if(a.compareTo(b)==0){
            System.out.println("一样高");
        }else{
            System.out.println(b.name+"工资高");
        }
    }
}

结果

B工资高

我们看 Comparable 接口还是比较简单的，只需要实现 comparaTo 方法即可，如果指定对象的类型与此对象不能进行比较，则引发 ClassCastException 异常（引入泛型可以规避这种情况）。

Comparator

先看一下源码

public class CmowerComparator implements Comparator<Cmower> {
    @Override
    public int compare(Cmower o1, Cmower o2) {
        return o1.getAge() - o2.getAge();
    }
}

实际上是要你重写类，以及如何比较的方法，举个例子。

public class Employee {
    private int age;
    private int salary;
    private String name;

    public Employee(int age, int salary, String name) {
        this.age = age;
        this.salary = salary;
        this.name = name;
    }

    public int getSalary() {
        return salary;
    }

    public String getName() {
        return name;
    }

    public static void main(String[] args) {
        Employee a = new Employee(18, 100, "A");
        Employee b = new Employee(18, 101, "B");
        Employee c = new Employee(18, 99, "C");
        List<Employee> list = new ArrayList<>();
        list.add(a);
        list.add(b);
        list.add(c);

        list.sort(new EmployeeComparator());

        for(Employee i:list){
            System.out.println(i.getName());
        }
    }
}

class EmployeeComparator implements Comparator<Employee>{
    @Override
    public int compare(Employee o1, Employee o2) {
        return o1.getSalary() - o2.getSalary();
    }
}

结果

C
A
B

显然，已经按照工资的顺序排列了。

Comparator 的具体作用就是判断大小的基础上应用如排序方法。

二者区别

  • 一个类实现了 Comparable 接口，意味着该类的对象可以直接进行比较（排序），但比较（排序）的方式只有一种，很单一。

  • 一个类如果想要保持原样，又需要进行不同方式的比较（排序），就可以定制比较器（实现 Comparator 接口）。

  • Comparable 接口在 java.lang 包下，而 Comparator 接口在 java.util 包下，算不上是亲兄弟，但可以称得上是表（堂）兄弟。

4. Iterable 和 Iterator

先看源码

public interface Iterable<T> {
  Iterator<T> iterator();
}

public interface Iterator<E> {
  boolean hasNext();
  E next();
  void remove();
}

Interable 只是返回了 Intertor 的一个实例，原因是实现了 Iterable 的类可以在实现多个 Iterator 内部类，例如 LinkedList 中的 ListItr 和 DescendingIterator 两个内部类，就分别实现了双向遍历和逆序遍历。通过返回不同的Iterator实现不同的遍历方式，这样更加灵活。如果把两个接口合并，就没法返回不同的 Iterator 实现类了。

上个代码，直观地展示一下实现了 Iterable 的类如何通过返回不同的 Iterator 从而实现不同的遍历方式。 MutilIterator 实现了三种迭代器，分别是默认的前向迭代器，反向迭代器和随机迭代器。主函数中分别调用了三种迭代器进行遍历。

public class MutilIterator implements Iterable<String>{

    private String[] words = "May I get offers this summer.".split(" ");

    // 默认迭代器，前向遍历
    @Override
    public Iterator<String> iterator() {
        return new Iterator<String>() {
            private int index = 0;
            @Override
            public boolean hasNext() {
                return index < words.length;
            }

            @Override
            public String next() {
                return words[index++];
            }

        };
    }

    // 反向迭代器
    public Iterable<String> reverseItertor(){
        return new Iterable<String>() {
            @Override
            public Iterator<String> iterator() {
                return new Iterator<String>() {
                    private int index = words.length - 1;
                    @Override
                    public boolean hasNext() {
                        return index>-1;
                    }

                    @Override
                    public String next() {
                        return words[index--];
                    }
                };
            }
        };
    }

    //随机迭代器，注意这里不是创建一个新的Iterator，而是返回了一个打乱的List中的迭代器
    public Iterable<String> randomized(){
        return new Iterable<String>() {
            @Override
            public Iterator<String> iterator() {
                List<String> shuffled = new ArrayList<>(Arrays.asList(words));
                Collections.shuffle(shuffled, new Random(47));
                return shuffled.iterator();
            }
        };
    }

    public static void main(String[] args) {
        MutilIterator mi = new MutilIterator();

        for(String i:mi){
            System.out.print(i + " ");
        }
        System.out.println();

        for(String i:mi.reverseItertor()){
            System.out.print(i + " ");
        }
        System.out.println();

        for(String i:mi.randomized()){
            System.out.print(i + " ");
        }
    }
}

以上就是三个常见的迭代器的用法。
